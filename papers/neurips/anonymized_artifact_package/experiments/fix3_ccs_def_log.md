# Fix 3 Log: CCS Definition Consistency

Date: 2026-04-29

Issue: the submission text described CCS as the mean pairwise similarity among
retrieved passages, while the original implementation defines CCS as the mean
minus the standard deviation of off-diagonal pairwise cosine similarities.

Decision: restore the original definition in the paper. No package version bump
is required because the code and released retriever behavior already use the
original mean-minus-standard-deviation definition.

Verification:

- `experiments/fix_01_causal_matched_pairs.py::_ccs_from_unit_embeddings`
  returns `upper.mean() - upper.std()`.
- `src/hcpc_v2_retriever.py` and `src/ccs_gate_retriever.py` also document and
  compute CCS as mean minus standard deviation.
- The matched-similarity construction in
  `data/revision/fix_01/matched_pairs.csv` was generated by the Fix 1 script
  using this definition.

Result: the null matched-similarity conclusion reported in the paper is already
the result under the original CCS definition. The paper now reports that
